# Contributing Guidelines

Thank you for taking the time to contribute.
We welcome code, documentation, bug reports, and ideas.

---

## Contribution Flow

1. Fork the repository

2. Create a branch using Git Flow naming:

```text
feature/short-description
bugfix/issue-number
hotfix/critical-fix
```

---

## Making Changes

- Write clean and readable code
- Follow existing conventions
- Keep changes focused

---

## Commit Message Convention

```text
feat(auth): add login validation
fix(api): handle null response
docs(readme): update setup steps
```

---

## Testing

- Add tests where applicable
- Ensure all existing tests pass

---

## Issues

- Search before creating a new issue
- Use the correct issue template
- Be clear and reproducible

---

## Pull Requests

- Use the PR template
- Link issues using Fixes #123
- Ensure CI checks pass
- Request review only when ready

---

## Contributor Credit

All contributions are credited automatically through GitHub history.

- Merged Pull Requests record authorship
- Commits and reviews are publicly visible
- No manual updates are required to receive credit

Please do not submit Pull Requests solely to add your name to files.

## Support

For questions, use Discussions or comment on the relevant issue.
